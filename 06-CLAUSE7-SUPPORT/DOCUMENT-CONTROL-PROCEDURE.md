# Document Control Procedure

**Document ID:** PIMS-SUP-004
**Version:** 1.0 | **Date:** 2025 | **Owner:** DPO

---

## Purpose

This procedure establishes how [Organisation Name] creates, approves, maintains, controls, and disposes of PIMS documented information, in accordance with ISO/IEC 27701:2025 Clause 7.5 (extension).

---

## Document Lifecycle

### 1. Creation
- All new PIMS documents must be initiated by the document owner
- Use the approved PIMS document templates (available on [intranet/SharePoint])
- Assign a unique Document ID using the format: PIMS-[SECTION]-[NUMBER] (e.g., PIMS-POL-001)
- Complete the document header: ID, version, date, owner, classification, review date

### 2. Review and Approval

| Document Type | Reviewer | Approver |
|---|---|---|
| Policy | DPO | CEO / CISO |
| Procedure | Privacy Team | DPO |
| Register / Log | Privacy Team / Owner | DPO |
| Template | Privacy Team | DPO |
| Worked Example | DPO | DPO |

### 3. Publication
- Approved documents are published to the PIMS document repository: [SharePoint/intranet path]
- Previous versions are archived in the document version history folder
- The PIMS Document Master List (PIMS-SUP-005) must be updated within 5 working days

### 4. Periodic Review
All PIMS documents must be reviewed at the following intervals:

| Document Type | Review Frequency | Trigger for Immediate Review |
|---|---|---|
| Policies | Annual | Regulatory change; significant organisational change |
| Procedures | Annual | Process change; audit finding; incident |
| Registers / Logs | As required | New entries; quarterly for summary review |
| Risk Register | Quarterly | New risk; breach; project |
| ROPA | Quarterly | New processing activity; change to existing |

### 5. Version Control

| Version Type | When Used | Format |
|---|---|---|
| Major (1.0, 2.0) | Significant content change; full re-approval | X.0 |
| Minor (1.1, 1.2) | Housekeeping; clarification; minor updates | X.Y |

Version history must be maintained within each document.

### 6. Disposal and Archiving
- Superseded documents are moved to the archive folder with the suffix [ARCHIVED-YYYY-MM-DD]
- Documents are retained for a minimum of 3 years after supersession
- Physical copies must be shredded using cross-cut shredders

## Document Classification

| Classification | Description | Access |
|---|---|---|
| Public | External-facing documents (Privacy Notice, Cookie Policy) | Anyone |
| Internal | Internal PIMS documents and procedures | All staff |
| Confidential | Sensitive documents (Risk Register, DPIA, ROPA) | Privacy Team, DPO, CISO, Management |
| Restricted | Highly sensitive (breach records, regulatory correspondence) | DPO, Legal, Senior Management only |
